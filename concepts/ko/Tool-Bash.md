# Bash

## 정의

셸 명령을 실행하며, 선택적 타임아웃 설정을 지원합니다. 작업 디렉토리는 명령 간에 유지되지만, 셸 상태 (환경 변수 등)는 유지되지 않습니다.

## 파라미터

| 파라미터 | 타입 | 필수 | 설명 |
|----------|------|------|------|
| `command` | string | 예 | 실행할 bash 명령 |
| `description` | string | 아니오 | 명령의 간단한 설명 |
| `timeout` | number | 아니오 | 타임아웃 (밀리초), 최대 600000, 기본값 120000 |
| `run_in_background` | boolean | 아니오 | 백그라운드에서 실행 여부 |

## 사용 시나리오

**적합한 경우:**
- git 작업 (commit, push, branch 등)
- npm/yarn 등 패키지 관리 명령
- docker 작업
- 컴파일, 빌드 명령
- 디렉토리 내용 목록 표시 (`ls`)
- 기타 셸 실행이 필요한 시스템 명령

**적합하지 않은 경우:**
- 파일 읽기 — Read를 사용해야 함
- 파일명 검색 — Glob를 사용해야 함
- 파일 내용 검색 — Grep를 사용해야 함
- 파일 편집 — Edit를 사용해야 함
- 파일 쓰기 — Write를 사용해야 함
- 사용자에게 정보 출력 — 응답 텍스트에 직접 출력
- 장시간 실행 프로세스 (dev server, watch 모드) — 사용자에게 수동 실행 권장

## 주의사항

- 공백이 포함된 경로는 큰따옴표로 감싸야 함
- 출력이 30000자를 초과하면 잘림
- 백그라운드 실행 명령은 TaskOutput으로 결과 획득
- 가능한 절대 경로를 사용하고 `cd`를 피할 것
- 독립적인 명령은 여러 Bash를 병렬 호출 가능
- 의존 관계가 있는 명령은 `&&`로 연결
- 셸 환경은 사용자의 profile (bash 또는 zsh)에서 초기화

## cc-viewer에서의 의의

Bash 호출은 요청 로그에서 `tool_use` (명령 포함)와 `tool_result` (출력 포함) content block 쌍으로 나타납니다. 명령 실행 출력은 모델의 작업 행동 분석에 사용할 수 있습니다.
