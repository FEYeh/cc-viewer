# Bash

## Визначення

Виконує shell-команду з підтримкою необов'язкового налаштування тайм-ауту. Робочий каталог зберігається між командами, але стан shell (змінні середовища тощо) не зберігається.

## Параметри

| Параметр | Тип | Обов'язковий | Опис |
|----------|-----|--------------|------|
| `command` | string | Так | Bash-команда для виконання |
| `description` | string | Ні | Короткий опис команди |
| `timeout` | number | Ні | Тайм-аут (мілісекунди), максимум 600000, за замовчуванням 120000 |
| `run_in_background` | boolean | Ні | Чи запускати у фоновому режимі |

## Сценарії використання

**Підходить для:**
- Операції git (commit, push, branch тощо)
- Команди менеджерів пакетів npm/yarn
- Операції docker
- Команди компіляції та збірки
- Перегляд вмісту каталогу (`ls`)
- Інші системні команди, що потребують виконання в shell

**Не підходить для:**
- Читання файлів — слід використовувати Read
- Пошук за іменем файлу — слід використовувати Glob
- Пошук вмісту файлів — слід використовувати Grep
- Редагування файлів — слід використовувати Edit
- Запис файлів — слід використовувати Write
- Виведення інформації користувачу — виводити безпосередньо в тексті відповіді
- Довготривалі процеси (dev server, режим watch) — рекомендується запускати вручну

## Примітки

- Шляхи з пробілами повинні бути обгорнуті подвійними лапками
- Вивід понад 30000 символів буде обрізано
- Результати фонових команд отримуються через TaskOutput
- Бажано використовувати абсолютні шляхи, уникаючи `cd`
- Незалежні команди можна викликати паралельно кількома Bash-викликами
- Залежні команди з'єднуються через `&&`
- Середовище shell ініціалізується з профілю користувача (bash або zsh)

## Значення в cc-viewer

Виклики Bash відображаються в журналі запитів як пара content block `tool_use` (містить команду) та `tool_result` (містить вивід). Вивід виконання команди може використовуватися для аналізу операційної поведінки моделі.
