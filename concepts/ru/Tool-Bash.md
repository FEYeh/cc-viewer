# Bash

## Определение

Выполняет команды shell с опциональной настройкой тайм-аута. Рабочий каталог сохраняется между командами, но состояние shell (переменные окружения и т.д.) не сохраняется.

## Параметры

| Параметр | Тип | Обязательный | Описание |
|------|------|------|------|
| `command` | string | Да | Команда bash для выполнения |
| `description` | string | Нет | Краткое описание команды |
| `timeout` | number | Нет | Тайм-аут (миллисекунды), макс. 600000, по умолчанию 120000 |
| `run_in_background` | boolean | Нет | Запускать ли в фоновом режиме |

## Сценарии использования

**Подходящее применение:**
- Операции git (commit, push, branch и т.д.)
- Команды менеджеров пакетов npm/yarn
- Операции docker
- Команды компиляции и сборки
- Просмотр содержимого каталогов (`ls`)
- Другие системные команды, требующие выполнения в shell

**Неподходящее применение:**
- Чтение файлов — следует использовать Read
- Поиск имён файлов — следует использовать Glob
- Поиск содержимого файлов — следует использовать Grep
- Редактирование файлов — следует использовать Edit
- Запись файлов — следует использовать Write
- Вывод информации пользователю — непосредственно в тексте ответа
- Долго работающие процессы (dev server, режим watch) — рекомендуется ручной запуск пользователем

## Примечания

- Пути, содержащие пробелы, должны быть заключены в двойные кавычки
- Вывод, превышающий 30000 символов, будет обрезан
- Результаты команд, запущенных в фоне, получают через TaskOutput
- Следует использовать абсолютные пути, избегать `cd`
- Независимые команды могут вызываться параллельно в нескольких Bash
- Команды с зависимостями связываются через `&&`
- Среда shell инициализируется из профиля пользователя (bash или zsh)

## Значение в cc-viewer

Вызовы Bash в логах запросов отображаются как пары блоков content `tool_use` (содержащие команду) и `tool_result` (содержащие вывод). Вывод выполнения команды может использоваться для анализа поведения модели.
