# EnterPlanMode

## التعريف

تحويل Claude Code إلى وضع التخطيط، لاستكشاف قاعدة الكود وتصميم الحلول قبل التنفيذ.

## المعاملات

لا توجد معاملات.

## سيناريوهات الاستخدام

**مناسب للاستخدام:**
- تنفيذ ميزات جديدة — تتطلب قرارات معمارية
- وجود عدة حلول ممكنة — تتطلب اختيار المستخدم
- تعديلات الكود تؤثر على السلوك أو البنية الحالية
- تغييرات متعددة الملفات — قد تشمل أكثر من 2-3 ملفات
- متطلبات غير واضحة — تحتاج استكشافاً أولاً لفهم النطاق
- تفضيلات المستخدم مهمة — يمكن أن يكون للتنفيذ عدة اتجاهات معقولة

**غير مناسب للاستخدام:**
- إصلاحات سطر واحد أو أسطر قليلة (أخطاء إملائية، أخطاء واضحة)
- المستخدم أعطى تعليمات محددة جداً
- مهام البحث/الاستكشاف البحتة — يجب استخدام Task (نوع Explore)

## السلوك في وضع التخطيط

بعد الدخول في وضع التخطيط، يقوم Claude Code بـ:
1. استخدام أدوات Glob وGrep وRead لاستكشاف قاعدة الكود بعمق
2. فهم الأنماط والبنية المعمارية الحالية
3. تصميم خطة التنفيذ
4. تقديم الخطة للمستخدم للموافقة
5. استخدام AskUserQuestion للتوضيح عند الحاجة
6. الخروج عبر ExitPlanMode عندما تكون الخطة جاهزة

## ملاحظات

- تتطلب هذه الأداة موافقة المستخدم للدخول في وضع التخطيط
- إذا لم تكن متأكداً من الحاجة للتخطيط، فضّل التخطيط — المواءمة المسبقة أفضل من إعادة العمل

## الأهمية في cc-viewer

يظهر استدعاء EnterPlanMode في سجل الطلبات ككتلة محتوى `tool_use`. بعد الدخول في وضع التخطيط، تتكون تسلسلات الطلبات عادة من استدعاءات أدوات استكشافية (Glob، Grep، Read) بشكل رئيسي، حتى يتم استدعاء ExitPlanMode.
